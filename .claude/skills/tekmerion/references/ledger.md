# Tracking artifacts: candidates, ledger, verification report

Three files beside the draft. The candidates file is generated and never
hand-edited; the ledger is curated; the verification report is produced at the
end and regenerated until clean. Together they make selection, coverage, and
reference integrity auditable at long-form scale, and they are the state that
survives across sessions.

## 1. `<term>_candidates.tsv` (generated, exhaustive, untouched)

One row per corpus **unit** matching the word-family, whole corpus, no
filters. Generated by a single SQL query; regenerate and diff on every resumed
session.

Header comment lines (before the TSV header row):

```
# term: <headword and word-family, privatives included>
# query: <the exact SQL, one line or quoted block>
# run: <ISO timestamp>
# counts: units=<n> tokens=<n> authors=<n>
```

Columns (tab-separated):

```
unit_id  wid  ref  author  affiliation  period  domain  format  fl_from  fl_to  token_count  length
```

All values straight from the DB at generation time. `token_count` is the
number of word-family tokens inside the unit (occurrence count), kept distinct
from the row itself: a token count is never an attestation count.

## 2. `<term>_ledger.md` (curated)

### Header block

- controlling question and hinge (as confirmed at the Phase 1 gate)
- complete-coverage set: the exact DB predicate
- trajectory type
- candidates file it derives from (name + run timestamp)

### Table

One row per unit that is a **station** or **secondary**, plus one row per
row-level **set-aside** (mandatory inside the complete-coverage set). Group
set-asides outside the coverage set are recorded as single lines under the
table, each with its group definition and reason, such that every candidates
row is covered by exactly one row or group line.

| # | unit_id | wid | ref | author | station_type | disposition | reason | verified | status |

- **#**: trajectory position (Phase 3). Set-asides carry none.
- **unit_id**: `eulogikon.units.id`, the verification anchor. The ref string
  is for humans; quotations are checked against the unit id.
- **station_type**: STYLE.md vocabulary: original speech, dialogue, medical
  treatise, doxography, testimonium, fragment-in-later-author, scholium,
  lexicon, commentary, hostile report, letter, historiography. Extend only
  from the evidence.
- **disposition**: `station` | `secondary` | `set-aside`.
- **reason**: required for every set-aside; inside the complete-coverage set
  the only valid values are `proper-name`, `duplicate-station of <unit_id>`,
  `corrupted-unit`, `quotes <unit_id>`.
- **verified**: the per-station verification record, filled at draft time
  (Phase 4.8): `Q` when every quoted span matched text_greek by substring,
  `C` when title/wid/ref/URL were resolved by query this session, `H:<source>`
  for any hand-supplied fact in the block. A station reads e.g. `Q C` or
  `Q C H:SVF-iii-731`. Empty means unverified: not publishable.
- **status**: `pending` | `drafted` | `checked` (verification report clean).

### Provenance marks

Any fact in the draft that is neither a QUOTE nor freshly-queried CATALOG is
HAND: it appears in the ledger row's `verified` column with its source, and in
the piece's caveats where it carries weight. The survey JSON is never a valid
HAND source: it is discovery only.

## 3. `<term>_verification.md` (produced)

Written in Phase 6, regenerated until clean. Sections:

1. **Quote integrity**: one line per station: unit_id, each quoted Greek
   segment (or its first/last words for long segments), match pass/fail.
2. **Coverage**: totals reconciling the candidates file: stations + secondaries
   + row set-asides + group set-asides = candidate rows. Any remainder is a
   failure.
3. **Catalog integrity**: each count, density, and table printed in the piece,
   with the query that reproduces it and the value obtained now.
4. **HAND register**: every hand-supplied fact, source, and where the piece
   discloses it.
5. **Prose scans**: blacklist result (live query, zero hits or listed
   exceptions with justification), U+2014 count (must be zero).

A check with no line here did not happen.

## Duplicate stations

The corpus can hold the same sentence at two addresses: a fragment-collection
work (a von Arnim volume under the fragmentary author's wid) and the quoting
source's own text (e.g. Diogenes Laertius). One witness, two stations. Rule:
the **source author's unit is the station** (the witness that actually
survives); the collection unit is `set-aside: duplicate-station of <unit_id>`;
the collection's fragment number is a HAND note in the station's citation or
commentary. Never count the pair as two attestations in any table.

## Corrupted or unsegmented units

A unit whose text concatenates many passages under one ref (anomalous length,
internal citations) cannot supply a citation from its ref field. Mark it
`corrupted-unit`; locate the passage inside it and supply the locus by HAND
from the internal apparatus; report the defect in the piece's
corpus-boundaries section; flag it upstream to the corpus project.
